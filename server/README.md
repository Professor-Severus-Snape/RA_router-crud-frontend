# Backend

Серверная часть приложения «Посты с реализацией роутинга».  
Описание проекта доступно в [главном README](../README.md).

## Назначение

Backend предоставляет REST API для работы с постами.

Обрабатывает HTTP-запросы, выполняет валидацию данных и возвращает JSON-ответы.

## API

- **GET /ping** — проверка доступности сервера
- **GET /posts** — получение списка постов
- **GET /posts/:id** — получение поста по id
- **POST /posts** — создание нового поста
- **PATCH /posts/:id** — обновление поста
- **DELETE /posts/:id** — удаление поста

## Особенности

- Полный CRUD-цикл для постов
- Валидация входных данных
- Корректные HTTP-статусы (200, 201, 204, 400, 404)
- Хранение данных в памяти (без базы данных)

## Архитектура

- REST API на Express
- Данные хранятся в массиве (in-memory)
- Автоинкремент id

## Технологии

- Node.js
- Express
- CORS
- REST API
