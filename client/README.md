# Frontend

Клиентская часть приложения «Посты с реализацией роутинга».  
Общее описание доступно в [главном README](../README.md).

## Назначение

Frontend реализует одностраничное приложение (SPA) для работы с постами через REST API.

Основной акцент — на клиентской маршрутизации, UX при загрузке и работе с асинхронными запросами.

## Особенности

- Проверка доступности сервера перед инициализацией приложения (`/ping`)
- Отложенное отображение loader без "мигания"
- Клиентская маршрутизация через React Router
- Работа с динамическими маршрутами (`/posts/:id`)
- Разделение сценариев: список, просмотр, создание, редактирование, удаление

## Архитектура

- Роутинг построен на React Router с поддержкой вложенных маршрутов
- Каждая страница изолирует свою бизнес-логику (список, просмотр, создание, редактирование)
- Общий Layout используется как базовый шаблон интерфейса
- Инициализация приложения (ping + loader) сосредоточена в корневом компоненте `App`
- Все сетевые запросы выполняются через Fetch API

## Работа приложения

### Инициализация

1. При запуске выполняется запрос `/ping`
2. Приложение отображается только после ответа сервера
3. При задержке показывается Loader (через 500ms)

### Маршруты

- `/` — список постов  
- `/posts/new` — создание поста  
- `/posts/:id` — просмотр поста  
- `/posts/update/:id` — редактирование  

### Поведение

- Все переходы выполняются без перезагрузки страницы
- После CRUD-операций происходит обновление данных или редирект
- Ошибки маршрутизации обрабатываются через `PageError`
